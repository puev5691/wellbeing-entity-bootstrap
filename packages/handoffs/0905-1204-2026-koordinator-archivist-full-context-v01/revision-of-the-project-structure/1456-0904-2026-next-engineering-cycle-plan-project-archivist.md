# План следующего инженерного цикла развития Архивариуса проекта Благополучие

Дата: 2026-04-09
Статус: рабочий план цикла
Контур: пакет текущего чата
Назначение: зафиксировать ближайший инженерный цикл развития MVP Архивариуса после первой рабочей публикации и первых реальных прогонов

## 1. Основание для нового цикла

К моменту начала этого цикла подтверждено, что Архивариус уже умеет:
- создавать SQLite-базу;
- индексировать файлы;
- искать по имени;
- искать по содержимому;
- распознавать пакет чата;
- работать на реальном пакете чата;
- существовать как опубликованный репозиторий.

Следовательно, следующий цикл должен усиливать не базовую жизнеспособность, а проектную полезность.

## 2. Главный приоритет цикла

Главный приоритет:
научить Архивариуса понимать `.wb-copy-map.tsv` не просто как файл, а как карту раскладки.

## 3. Что это должно дать

После реализации этого шага Архивариус должен начать помогать отвечать на вопросы:

- какие файлы пакета чата предполагается раскладывать дальше;
- в какие канонические каталоги они должны попадать;
- какие карты раскладки существуют в пакете;
- как связать происхождение файла и его предполагаемое каноническое место.

## 4. Минимальный состав работ цикла

1. определить минимальный формат разбора `.wb-copy-map.tsv`;
2. реализовать парсер карты;
3. добавить хранение или выдачу результатов разбора;
4. добавить CLI-команду для показа карты раскладки;
5. проверить на текущем реальном пакете чата.

## 5. Минимальный критерий успеха

Цикл можно считать успешным, если Архивариус будет способен:
- обнаружить `.wb-copy-map.tsv`;
- прочитать его строки;
- показать, какие файлы и куда должны быть разложены;
- сделать это на текущем пакете чата.

## 6. Что не является задачей этого цикла

На этом этапе не требуется:
- автоматически копировать файлы;
- строить сложную систему статусов;
- вводить GUI;
- расширять охват на всё хранилище;
- делать сложную онтологию канонических связей.

## 7. Итог

Следующий инженерный цикл сосредотачивается на превращении Архивариуса:
из локального индексатора и поисковика
в инструмент, который начинает понимать логику раскладки файлов из чатовых пакетов в канонические контуры проекта.
