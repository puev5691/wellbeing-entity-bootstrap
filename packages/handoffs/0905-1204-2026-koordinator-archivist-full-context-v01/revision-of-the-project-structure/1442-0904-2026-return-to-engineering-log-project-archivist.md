# Журнал возврата к инженерной работе над Архивариусом проекта Благополучие

Дата: 2026-04-09
Статус: журналная запись
Контур: пакет текущего чата
Назначение: зафиксировать решение о возврате от литературной ветки к основной инженерной задаче развития Архивариуса

## 1. Основание для записи

После успешного:
- проектирования Архивариуса;
- сборки MVP-каркаса;
- прохождения smoke test;
- ограниченного реального прогона;
- распознавания реального пакета чата;
- первой выкладки репозитория в GitHub;

а также после развёртывания литературной ветки по теме Архивариуса, было принято решение вернуться к основной инженерной задаче.

## 2. Причина возврата

Причина возврата проста:
литературная работа важна и полезна для медиаконтура, но сама программа Архивариус ещё не доведена до следующего practically significant уровня полезности.

На текущем этапе у Архивариуса уже есть:
- индексирование;
- поиск по имени;
- поиск по содержимому;
- распознавание пакета чата;
- локальная SQLite-база;
- репозиторный контур.

Но отсутствуют следующие важные усиления:
- осмысленная работа с `.wb-copy-map.tsv`;
- более выразительный вывод результатов;
- дальнейшее развитие логики пакетной раскладки;
- расширение реального охвата хранилища.

## 3. Что признано следующим инженерным приоритетом

Следующим инженерным приоритетом признано развитие MVP в сторону большей проектной полезности.

Наиболее разумный следующий шаг:
научить Архивариуса не только видеть `.wb-copy-map.tsv` как файл, но и понимать его как карту раскладки.

Это должно позволить:
- связывать файлы пакета чата с каноническими каталогами;
- видеть предполагаемую судьбу файлов;
- сделать Архивариуса не только поисковиком, но и помощником по раскладке.

## 4. Состояние на момент записи

На момент записи:
- литературная ветка зафиксирована;
- инженерная ветка остаётся основной;
- проект Архивариуса уже имеет первую рабочую форму;
- следующая задача определена.

## 5. Итог

Настоящая запись фиксирует переход:
от временного литературного расширения темы
обратно к основному инженерному циклу развития Архивариуса.

Следующий рабочий шаг:
короткий инженерный план следующего цикла и затем правка кода под поддержку `.wb-copy-map.tsv`.
