## 1. Разрешённые состояния шага

Каждый шаг должен находиться только в одном из состояний:

- script_issued
- script_executed
- artifact_confirmed

## 2. Определения

### script_issued
Сущность выдала shell-скрипт, но оператор ещё не подтвердил его выполнение.

### script_executed
Оператор выполнил скрипт, но ещё не подтвердил, что целевой артефакт действительно существует и соответствует ожиданию.

### artifact_confirmed
Оператор подтвердил артефакт фактическими данными файловой системы.

## 3. Что считается подтверждением артефакта

Подтверждением артефакта считаются только фактические команды, например:

- ls -l <полный_путь_к_файлу>
- sed -n '1,40p' <полный_путь_к_файлу>
- find <каталог> -maxdepth ... | sort
- cat .wb-copy-map.tsv

## 4. Главный запрет

Нельзя выдавать следующую фазу как update существующего файла, если состояние предыдущего шага не доведено до artifact_confirmed.

## 5. Правило для Координатора

Координатор не имеет права предполагать фактическое существование файла только потому, что раньше был выдан скрипт на его создание.

## 6. Правило для дочерних Сущностей

Если шаг зависит от существования файла, дочерняя Сущность должна:
- либо работать в режиме безопасного создания/восстановления;
- либо явно требовать подтверждения существования артефакта.

## 7. Правило для оператора

Оператор не обязан помнить состояние шага по памяти.
Оператор должен фиксировать подтверждение артефакта выводом команд файловой системы.
