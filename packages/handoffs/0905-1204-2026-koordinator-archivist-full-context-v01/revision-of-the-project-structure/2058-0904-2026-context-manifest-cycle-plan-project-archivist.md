# План цикла развития Архивариуса по манифесту контекста для новой Сущности

Дата: 2026-04-09
Статус: рабочий план цикла
Контур: пакет текущего чата
Назначение: зафиксировать следующий цикл, в котором Архивариус начнёт собирать рекомендованный манифест файлов для передачи контекста новой Сущности или нового специализированного чата

## 1. Основание для цикла

Архивариус уже умеет:
- индексировать файлы;
- искать;
- различать типы и часть статусов;
- работать с картой раскладки;
- проверять покрытие;
- рекомендовать пополнение карты;
- готовить candidate и append patch;
- давать статусный отчёт по пакету.

Следовательно, следующий логический шаг:
начать помогать не только с внутренним порядком пакета, но и с передачей контекста дальше.

## 2. Главная цель цикла

Главная цель:
научить Архивариуса собирать манифест контекста для новой Сущности.

## 3. Минимальный состав манифеста

На первом этапе манифест должен различать хотя бы такие разделы:

- package_readme
- engineering_core
- logs_and_decisions
- literary_context
- manual_review

## 4. Практический смысл результата

Практический смысл результата:
снизить объём ручного выбора файлов при запуске нового чата, handoff или дочерней Сущности.

## 5. Итог

Этот цикл должен стать переходом:
от внутренней ревизии пакета
к управляемой передаче контекста следующему рабочему контуру.
