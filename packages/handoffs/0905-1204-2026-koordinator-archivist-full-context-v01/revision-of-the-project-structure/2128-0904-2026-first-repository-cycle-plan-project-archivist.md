# План первого репозиторного цикла развития Архивариуса

Дата: 2026-04-09
Статус: рабочий план цикла
Контур: пакет текущего чата
Назначение: зафиксировать первый цикл развития Архивариуса в сторону репозиторного режима для подготовки контекста кодовым Сущностям

## 1. Основание для цикла

Архивариус уже развивается как инструмент для работы с пакетами чатов, журналами, markdown-документами и картами раскладки.

Следующий значимый шаг:
сделать его полезным и для работы с репозиториями исходного кода.

Это особенно важно для задач, где новым Сущностям придётся работать с кодом при ребрендинге и переработке исходников криптоплатформы.

## 2. Главная цель цикла

Главная цель:
научить Архивариуса делать первичный обзор репозитория и готовить манифест для кодовой Сущности.

## 3. Что должен уметь первый репозиторный режим

На первом этапе Архивариус не должен притворяться анализатором кода уровня божественного суда.

Ему достаточно уметь:

- распознавать, что каталог является git-репозиторием
- выделять ключевые файлы и каталоги
- собирать обзор структуры репозитория
- подготавливать минимальный манифест контекста для кодовой Сущности

## 4. Какие элементы нужно различать в репозитории

Минимально полезные категории:

- документация
- конфиги
- зависимости
- исходный код
- скрипты
- тесты
- артефакты сборки
- мусорные или вторичные каталоги

## 5. Какие файлы и признаки особенно важны

При первом репозиторном цикле особое внимание нужно уделять:

- `.git/`
- `README*`
- `CHANGELOG*`
- `pyproject.toml`
- `requirements.txt`
- `package.json`
- `docker-compose*`
- `Makefile`
- `src/`
- `app/`
- `docs/`
- `scripts/`
- `tests/`

## 6. Какой результат считать успешным

Успешным результатом первого репозиторного цикла следует считать:

- возможность указать путь к репозиторию
- получение обзорного манифеста репозитория
- выделение ключевых файлов и директорий
- формирование стартового набора контекста для кодовой Сущности

## 7. Практический смысл результата

Это позволит быстрее запускать специализированные Сущности для:

- анализа архитектуры репозитория
- подготовки к ребрендингу
- поиска точек входа
- понимания структуры конфигов и зависимостей
- передачи стартового кодового контекста без ручного блуждания по исходникам

## 8. Итог

Первый репозиторный цикл следует считать обязательным следующим расширением полезности Архивариуса после укрепления документно-пакетного режима.
