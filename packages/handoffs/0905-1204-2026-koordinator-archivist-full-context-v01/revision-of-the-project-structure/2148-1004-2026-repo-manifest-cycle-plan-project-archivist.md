# План цикла развития Архивариуса по первому репозиторному манифесту

Дата: 2026-04-10
Статус: рабочий план цикла
Контур: пакет текущего чата
Назначение: зафиксировать цикл, в котором Архивариус начнёт строить обзорный манифест репозитория для кодовой Сущности

## 1. Основание для цикла

Архивариус уже умеет работать с пакетами чатов, журналами, картами раскладки и статусами файлов.

Следующий логический шаг:
дать ему возможность готовить стартовый обзор репозитория исходного кода.

## 2. Главная цель

Главная цель:
получать из указанного пути к репозиторию краткий, но practically useful манифест для Сущности, которой предстоит работать с кодом.

## 3. Что должен показать первый манифест

На первом этапе манифест должен включать хотя бы:

- имя и путь репозитория
- является ли каталог git-репозиторием
- ключевые файлы документации
- ключевые файлы зависимостей и конфигов
- важные каталоги
- вероятные точки входа
- верхнеуровневые элементы, требующие ручного просмотра

## 4. Практический смысл результата

Это позволит быстрее запускать кодовые Сущности для:

- исследования архитектуры
- ребрендинга
- анализа точек входа
- понимания структуры проекта
- отбора стартового контекста без ручного блуждания

## 5. Итог

Этот цикл должен стать первым рабочим переходом Архивариуса от документно-пакетного режима к репозиторному режиму.
